Grading Page

Graphical and Interaction Design

  • It is obvious to a site visitor that all pages are part of the same site as they all contains repetition and same components.
  • Global navigation is accessible on all pages.
  • Prevents site visitor from getting lost using page titles and div tags (excluding reference and grading pages) highlighting/selecting the current page the user is on.
  • The site easy to navigate and each pages aren't overwhelmingly long.
  • Page titles (title tags) reflects the actual content of the page.
  • Able to display well across all major, modern browsers.

Content

  • Maps are provided and are related to the contents of the site.
  • Reference/Citation is provided for each of the photographs and images.
  • More than two sentences of content on each page.
  • Content is written for the purpose of attracting site visitors to visit Rochester area.

Code Design

  • According to professor, SSI is not needed.
  • Code is easy to read.
  • Code structure makes sense, such as, no HTML tables used for non-tabular data.
  • CSS file is commented.
  • HTML5 is used and all pages validated.
  • HTML5 semantic/structural elements are appropriately used.
  • Codes (including HTML and CSS) not written by myself aren't used.

Overall Contains

  • Design techniques is used according to CRAP.
  • Text and graphics make sense.
  • Provides interesting read and such for the content.
  • HTML5 and CSS3 are used.
  • No SSI needed.
  • A reference page.
  • A grading page.
